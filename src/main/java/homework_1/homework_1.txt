1. Как организовать клиент-серверное взаимодействие?

запустить на одном компьютере программу сервер отвечающую на запросы клиентов, 
на других компьютерах запустить программы клиенты (браузеры, почтовые клиенты...), которые посылают запросы серверу

2. Как и в каком виде передавать файлы?

- Формат: передавать в виде байтов, текста, json
- Размер: Или целиком или разбивать на байтовые блоки
- Можно использовать свой протокол передачи данных
- Можно исполльзовать протокол передачи данных, например TCP, FTP, UDP
- Так же можно предоставлять доступ к директории с файлом (шаринг)


3. Как пересылать большие файлы?

- Дробить (сегментировать данные) желательно при помощи протоколов, 
- zero copy


4. Как пересылать служебные команды?

- если передаем помимо объектов команды в виде json там может быть метаинформация с командами, которые можно парсить
- если передаем команды вместе с другими объектами можно использовать делимитер (метки) для отделения этих команд от других данных, 



5. Что хранить в базе данных?

- Либо ссылки на адреса и имена файлов каждого пользователя, 
- либо название файла и сами файлы в байтовом представлении blob


6. Как передавать структуру каталогов/файлов?

- в метаинформации к данным, либо отдельным сообщением
- в виде объекта класса Path


7. Какую библиотеку использовать для сетевого взаимодействия: java.io, java.nio, Netty?

- Любую с которой знаком либо располагаешь временем ознакомиться, но 
io при большом количестве подключений будет нагружать систему изза ого что на каждое соединение понадобится свой поток
nio вроде как сложный и запутанный
по этому похоже лучше netty
